# Looker Studio Deployment & Setup Guide

**Owner:** Daniel Rodriguez III
**Phase:** 4 (Visualization & Executive Reporting)

## 1. Prerequisites
* Active Google Cloud Account with access to the `driiiportfolio` project.
* BigQuery tables populated via the Phase 2 Python scripts.
* SQL views instantiated in BigQuery using scripts `01` through `03`.

## 2. Connecting the Data Source
1. Navigate to Looker Studio (lookerstudio.google.com).
2. Click **Create** > **Data Source**.
3. Select the **BigQuery** connector.
4. Navigate to **My Projects** > `driiiportfolio` > `aso_operations_2026`.
5. Select the materialized view generated by `01_revenue_deduplication.sql` and click **Connect**.
6. Repeat this process for the labor optimization and agentic commerce SQL views.
   * vw_daily_revenue_metrics
   * vw_hourly_labor_transactions_joined
   * vw_peak_hours_identified
   * vw_splh_analysis
   * vw_transactions_deduplicated

## 3. Configuring the Dashboard from JSON Schema
To bypass manual chart creation, utilize the provided `looker_studio_dashboard_export.json` schema layout.

1. Ensure all custom calculated fields match the aliases in the SQL queries (e.g., `net_revenue`, `redownload_ratio`).
2. Map the Data Sources in Looker Studio to the specific pages as dictated by the schema file.
   * Page 1 connects exclusively to the vw_daily_revenue_metrics view.
   * Page 2 connects to the vw_hourly_labor_transactions_joined view.
   * Page 3 connects to the vw_hourly_labor_transactions_joined view.
3. Apply a global date range filter set to **April 1, 2026 – April 30, 2026** to capture the full synthetic cohort and the pre/post Keyword Earthquake event.
